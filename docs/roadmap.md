# mebuki ロードマップ

調査日: 2026-04-25 / 対象バージョン: v2.13.0

---

## 現状サマリー

mebuki は J-QUANTS・EDINET の2API を束ね、CLI と MCP サーバーの両インターフェースから日本株の財務分析を行うツール。
コア機能（MCP 9ツール・多年度財務分析・XBRL 解析・ポートフォリオ管理）は動作しているが、**未完成の機能と技術的負債**が散在する。

## 優先度高（インパクト大）

### 1. LLM 統合の完成

**現状:** `llm/providers.py` に Gemini・Ollama の部分実装はあるが、Claude API プロバイダが未実装。`config.py` に "claude" が許容値として列挙されているが実体なし。Grok も同様。

**改良内容:**
- `llm/providers.py` に `ClaudeProvider` クラスを実装（Anthropic SDK 使用）
- トークン使用量・コスト推定の記録
- LLM 機能を CLI/MCP に配線（現状は `llm_provider = "none"` のまま）
- プロバイダ設定が無効な場合の起動時バリデーション

**期待効果:** mebuki 自体が Claude を使って分析コメントや要約を生成できるようになる

---

### 2. インタラクティブ TUI モードの実装

**現状:** `mebuki`（引数なし）でバナーを表示後にヘルプを出すだけ。questionary 依存はインストール済みだが未配線。`cmd_portfolio_interactive` など部品は存在する。

**改良内容:**
- `app/cli/main.py` にメインループを実装
- questionary を使ったメニュー: 銘柄検索 → 分析 → ファイリング → ポートフォリオ
- コンテキスト引き継ぎ（検索した銘柄コードをそのまま分析に渡す）

**期待効果:** コマンドを覚えなくてもツールを使えるユーザー導線が生まれる

---

### 3. ポートフォリオ高度化

**現状:** ポジション管理（追加・売却・削除）と総平均法コストは実装済み。損益計算・税務追跡・配当は未実装。

**改良内容:**
- 損益（実現・含み）計算の追加
- セクター配分・加重リターンの集計
- 配当履歴のトラッキング（`portfolio.json` のデータモデル拡張）
- FIFO/LIFO のロット管理オプション（現状は 総平均法のみ）

---

### 4. XBRL 解析の堅牢化

**現状:**
- US-GAAP の HTML パース処理が `gross_profit.py` と `xbrl_parser.py` で重複
- XBRL 抽出結果がキャッシュされないため毎回ダウンロード・解析が発生
- エラー時は警告ログのみで空文字列を返す（呼び出し側が気づけない）

**改良内容:**
- US-GAAP HTML パース処理を `xbrl_utils.py` に統合（DRY 化）
- 抽出済み XBRL の JSON キャッシュ追加（`cache/xbrl/` 以下）
- 抽出失敗時に戻り値で失敗理由を明示（error-handling 規約準拠）

---

## 優先度中（安定性・品質向上）

### 5. 非同期処理の最適化とプログレス表示

**現状:** EDINET ドキュメントのダウンロードがボトルネックで、10年分の分析で2〜3分かかることがある。長時間操作でプログレス表示がない。

**改良内容:**
- EDINET ダウンロードを並列化（現状はロック付き逐次処理）
- `rich.progress` または `tqdm` でプログレスバーを追加
- タイムアウト値を設定ファイルで変更可能にする（現状 `edinet_fetcher.py` で 30s ハードコード）

---

### 6. マスタデータの動的更新

**現状:** 銘柄リストは静的バンドル（assets 配下）。新規 IPO・市場変更・銘柄廃止が反映されない。

**改良内容:**
- `mebuki master-data update` コマンドの追加
- J-QUANTS API から最新の株式マスタを取得してローカルに保存
- キャッシュ同様の TTL 管理（例: 7日）

---

### 7. エラーハンドリングの統一

**現状:**
- J-QUANTS クライアント: 5回リトライ + 指数バックオフ + 429 対応
- EDINET クライアント: 3回リトライのみ
- CLI の analyze コマンドで `except Exception` の素のキャッチが複数箇所
- エラーメッセージが英語・日本語混在

**改良内容:**
- EDINET クライアントを J-QUANTS に揃える（retry 回数・バックオフ戦略）
- ユーザー向けメッセージを日本語に統一
- `except Exception` を具体的な例外型または戻り値パターン（error-handling 規約準拠）に置換

---

### 8. 設定バリデーション強化

**現状:** `config.json` に JSON スキーマ検証なし。無効な LLM プロバイダ名を書いても起動時にエラーにならない。

**改良内容:**
- `infrastructure/settings.py` に JSON スキーマ or Pydantic バリデーション追加
- API キーの存在チェックを起動時に実行するオプション（`mebuki config check`）
- キャッシュ設定変更時に既存キャッシュを無効化するか警告
