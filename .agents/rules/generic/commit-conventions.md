# コミットメッセージ規約

[Conventional Commits](https://www.conventionalcommits.org/) に準拠する。

## フォーマット

```
<type>: <summary>
```

- `type` は下表から選ぶ
- `summary` は英語・小文字始まり・命令形（動詞で始める）・末尾ピリオドなし

## type 一覧

| type | 用途 |
|---|---|
| `feat` | 新機能追加 |
| `fix` | バグ修正 |
| `refactor` | 動作を変えないコード整理 |
| `docs` | ドキュメントのみの変更 |
| `chore` | ビルド・バージョン・設定など（プロダクションコード非変更） |
| `test` | テストの追加・修正 |
| `perf` | パフォーマンス改善 |

## ルール

- 1コミット = 1つの論理的変更
- バージョンバンプは機能コミットと分離し、独立した `chore:` コミットにする
- スコープ（`feat(auth): ...` 形式）は使わない
